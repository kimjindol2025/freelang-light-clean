## 💡 주요 개념

### 고정점(Fixed Point)

```
정의:
  f(x) = x 를 만족하는 x

컴파일러 세계:
  - C 컴파일러를 C로 구현
  - 그 C 컴파일러로 자신을 컴파일 가능

FreeLang:
  - FreeLang 파서를 FreeLang으로 구현
  - 그 파서로 자신의 소스코드 파싱 가능

Phase 3:
  - parser.fl이 parser.fl 자신을 파싱할 수 있는가?
  - 성공 = 고정점 달성 = Bootstrapping의 증명
```

### Bootstrapping 단계

```
Phase 1: Lexer
  ✅ 토큰화 함수 (tokenize)
  ✅ 3개 파일 토큰화 검증

Phase 2: Parser
  ✅ 파싱 함수 (parse)
  ❌ 자신을 파싱하지 못함 (실행 검증 불가)

Phase 3: Self-Parse
  ⏳ parser.fl이 자신을 파싱할 수 있는가?
  ← 여기가 고정점

Phase 4: Code Generation
  → C 코드 생성기 (FreeLang으로 구현)
  → 자신의 코드를 C로 컴파일 가능

Phase 5: Full Self-Hosting
  → TypeScript 제거
  → 100% FreeLang 자체호스팅
```
